## 4. R6 commits & changes (NEW vs R5)

3 commit-uri: `c06c53f → 8163639 → 4d68dd0`.

### `c06c53f` — fix(license): tolerate anti-rollback clock drift up to 30 days

**Files**: `src-tauri/src/commands/license.rs` (+39 −2)

**De ce contează**: hard-fail-ul vechi `now < last_seen - 300` (5 minute) bloca app-ul orice timp când `license_last_seen_v2` era în viitor. Întâlnit azi: dev machine a primit drift de ~10 zile dintr-o eroare de injectare a unui trial. Single launch → cryptic "Licență invalidă" → singura cale era ștergerea manuală a tabelei `license` și a setting-urilor `license_*`.

**Comportament nou**:
- `drift ≤ 86400s (1 zi)`: silent clamp (setting-ul `license_last_seen_v2` e suprascris la `now` necondiționat la linia 381).
- `86400 < drift ≤ 2592000s (30 zile)`: `tracing::warn!`, clamp, app continuă.
- `drift > 30 zile`: `return Ok(false)` → user re-activează (presupune tamper real).

Două teste noi în `sec_tests`: `anti_rollback_tolerates_one_day_future_drift`, `anti_rollback_hard_fails_above_30_days`. Total teste: 55 → 57.

### `8163639` — feat(cross-platform): single-instance focus + modernize platform detect

**Files**: `src-tauri/Cargo.toml` (+1), `src-tauri/src/lib.rs` (+7), `src/lib/platform.ts` (+17 −6)

**De ce contează**: închide **EDGE-25** din auditul live R4 — dacă userul lansa app-ul de două ori (ex. din Dock + din Launchpad), două procese rulau în paralel `background/recurring.rs`, `background/spv.rs`, `poll_submitted_invoices` etc., generând auto-submit dublu și double-fetch SPV.

**Modificări**:
- `tauri-plugin-single-instance = "2"` adăugat în secțiunea `# Tauri plugins` Cargo.toml.
- `lib.rs`: pluginul wire-uit **primul** în `tauri::Builder::default()`. Closure: la a doua lansare, găsește `webview_window("main")` și apelează `unminimize() + show() + set_focus()`. Funcționează identic pe Mac (CFBundle uniqueness), Win (mutex named), Linux (DBus).
- `platform.ts`: vechiul `navigator.platform` (deprecated, fingerprint-vector) înlocuit cu `navigator.userAgentData.platform`. Fallback pe `navigator.userAgent` pentru WebView-uri vechi. **Semnătura publică (`isMac: boolean`, `fmtShortcut`) neschimbată** → toate call site-urile rămân stabile.

## 5. License blocker — post-mortem + runbook

### Root cause (incident azi)

Dev machine avea `license_last_seen_v2 = 1779995750` (≈ 8 iunie 2026, 9-10 zile în viitor față de `now`). Pre-R6, `commands/license.rs:368-378` făcea:

```rust
if let Some(ls_str) = get_setting(pool, "license_last_seen_v2").await? {
    let ls: i64 = ls_str.parse().unwrap_or(0);
    if now < ls.saturating_sub(300) {
        return Ok(false); // hard fail — license invalid
    }
}
```

`OnboardingGate.tsx` interpreta `Ok(false)` ca "trial expirat / tampered" → bloca complet app-ul. Singurul fix era manual: ștergere DB row + setări + restart.

### Fix-ul R6 (c06c53f)

Vezi secțiunea 4 — tiered drift cu `TOLERANCE_SILENT = 86400`, `TOLERANCE_HARD_FAIL = 2592000`. Acum un drift cauzat de DST, NTP correction după sleep, sau injecție trial buggy se auto-vindecă.

### Runbook — dacă reapare totuși un blocker dur (de ex. user pe build vechi sau drift > 30 zile)

```bash
# 1. Stop app
pkill -f "RoFactura\|efactura-desktop"

# 2. Backup + curăță starea licenței
cp ~/Library/Application\ Support/com.lucaris.efactura/data.db{,.backup-$(date +%Y%m%d-%H%M%S)}
sqlite3 ~/Library/Application\ Support/com.lucaris.efactura/data.db \
  "DELETE FROM license WHERE id=1; DELETE FROM settings WHERE key LIKE 'license_%';"
security delete-generic-password -s "ro.lucaris.efactura.trial.v1" -a "trial_status" 2>/dev/null

# 3. Re-acordă trial via Python script (păstrat la /tmp/grant_rofactura_trial.py)
python3 /tmp/grant_rofactura_trial.py
```

### Cum funcționează scriptul

`/tmp/grant_rofactura_trial.py` (sursă completă în Appendix, §11):

- Calculează `machine_id = sha256("{LocalHostName}||{user}||macos")[:24]`.
- Folosește `SECRET` care e **byte-exact** ce e definit în `src-tauri/build.rs:27` (XOR-obfuscat la compile time, dar valoarea clear ajunge în binar via macro-ul generat în `OUT_DIR/license_secrets.rs`).
- Compune `fingerprint = sha256(SECRET || email || mid || expires_at || tier)`.
- Inserează direct în SQLite: tabela `license` (id=1, tier=TRIAL, expires_at=now+14d), apoi `settings.license_fp_v2` + `settings.license_last_seen_v2`.
- Ocolește `start_trial()` cu validarea keychain-marker — utilizatorii reali îl primesc prin onboarding, scriptul e pentru dev unblock.

**ATENȚIE la version bump**: `src-tauri/build.rs:27` derivă XOR-salt-ul din `pkg_name@pkg_version`. La bump `0.1.0 → 0.2.0`, **salt-ul se schimbă** → fingerprint-uri vechi de TRIAL/SOLO devin invalide. Dev trebuie să re-ruleze scriptul după primul build pe noua versiune.

## 11. Appendix — Script unblock TRIAL

Sursa completă a `/tmp/grant_rofactura_trial.py` (păstrat self-contained în caz că `/tmp` e șters):

```python
#!/usr/bin/env python3
"""One-off: grant a fresh 14-day RoFactura TRIAL on this machine.

Uses the same INTEGRITY_SECRET embedded in build.rs to compute a fingerprint
that check_license_validity() will accept. Bypasses the keychain marker check
(which is the only thing start_trial() would block on).
"""
import hashlib, sqlite3, subprocess, time, pathlib, sys

EMAIL = "cristian@gmail.com"
TRIAL_DAYS = 14

# From src-tauri/build.rs (line 27) — 35 bytes
SECRET = b"RoF@ctura#2026!intgr1ty_K3y\xd4\x9a\x7f\x01\xbe\xc3v2"
assert len(SECRET) == 35, f"secret length {len(SECRET)} != 35"

DB = pathlib.Path.home() / "Library/Application Support/com.lucaris.efactura/data.db"
assert DB.exists(), f"DB not found at {DB}"


def machine_id() -> str:
    host = subprocess.check_output(["scutil", "--get", "LocalHostName"]).decode().strip()
    user = subprocess.check_output(["id", "-un"]).decode().strip()
    raw = f"{host}||{user}||macos".encode()
    return hashlib.sha256(raw).hexdigest()[:24]


def fingerprint(email: str, mid: str, expires_at: int, tier: str) -> str:
    h = hashlib.sha256()
    h.update(SECRET);            h.update(b"\x00")
    h.update(email.encode());    h.update(b"\x00")
    h.update(mid.encode());      h.update(b"\x00")
    h.update(str(expires_at).encode()); h.update(b"\x00")
    h.update(tier.encode())
    return h.hexdigest()


mid = machine_id()
now = int(time.time())
expires = now + TRIAL_DAYS * 86400
email_lower = EMAIL.strip().lower()
fp = fingerprint(email_lower, mid, expires, "TRIAL")

con = sqlite3.connect(DB)
cur = con.cursor()
cur.execute(
    "INSERT INTO license (id, tier, activated_at, expires_at, machine_id, email, last_validated_at) "
    "VALUES (1, 'TRIAL', ?, ?, ?, ?, ?) "
    "ON CONFLICT(id) DO UPDATE SET tier='TRIAL', activated_at=excluded.activated_at, "
    "expires_at=excluded.expires_at, machine_id=excluded.machine_id, "
    "email=excluded.email, last_validated_at=excluded.last_validated_at",
    (now, expires, mid, email_lower, now),
)
cur.execute(
    "INSERT INTO settings(key, value) VALUES(?, ?) ON CONFLICT(key) DO UPDATE SET value=excluded.value",
    ("license_fp_v2", fp),
)
cur.execute(
    "INSERT INTO settings(key, value) VALUES(?, ?) ON CONFLICT(key) DO UPDATE SET value=excluded.value",
    ("license_last_seen_v2", str(now)),
)
con.commit()
con.close()

print(f"✓ TRIAL granted")
print(f"  email:       {email_lower}")
print(f"  machine_id:  {mid}")
print(f"  now:         {now} ({time.ctime(now)})")
print(f"  expires_at:  {expires} ({time.ctime(expires)})")
print(f"  fp[:16]:     {fp[:16]}…")
```

**Pentru Windows / Linux**: schimbă `machine_id()` — folosește `os.environ['COMPUTERNAME']` + `getpass.getuser()` pentru Win sau `pathlib.Path('/etc/hostname').read_text().strip()` pentru Linux. `DB` path se schimbă conform §10.2. SECRET-ul rămâne identic câtă vreme `pkg_version` nu s-a schimbat.
